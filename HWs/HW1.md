# Задание
Система управления конференциями (Conference Management System)

---

## 1. Отсев

| Элемент backlog                               | Решение | Причина                           |
|-----------------------------------------------|--------:|-----------------------------------|
| Просмотр расписания онлайн                    | MVP     | Ключевой пользовательский сценарий |
| Управление докладами спикерами                | MVP     | Основная бизнес-функция           |
| Голосование за доклады                        | MVP     | Влияет на качество контента       |
| Уведомления об изменениях расписания          | MVP     | Критично для UX                   |
| Брендинг конференций                          | MVP     | Требование бизнеса                |
| Доступ к слайдам только для участников        | MVP     | Контроль доступа                  |
| Система оценки (web/email/SMS/phone)          | MVP+    | Можно упростить на старте         |
| SMS/телефон интеграции                        | MVP+    | Дорого и сложно                   |
| Обновления в реальном времени (websocket/push) | MVP     | Требование real-time UX [^1]      |
| ML-рекомендации докладов                      | Later   | Не входит в базовый scope         |
| Социальные функции (комментарии, чат)         | Later   | Не критично                       |
| Видеостриминг докладов                        | Later   | Не указано в требованиях          |

---

## 2. Scope Refinement

| Зона                | Входит в scope                                             | Не входит в scope      |
|---------------------|------------------------------------------------------------|------------------------|
| Web (Attendees)     | Просмотр расписания, голосование, доступ к слайдам, оценки | Социальная сеть        |
| Web (Speakers)      | CRUD докладов, загрузка материалов                         | Продвинутая аналитика  |
| Панель организатора | Управление конференциями, расписанием, уведомлениями       | Финансовый учет        |
| Backend             | Аутентификация, расписание, голосование, уведомления       | ML/AI                  |
| Уведомления         | Push/email (MVP), SMS                                      | Голосовые системы      |
| Хранилище           | Слайды, метаданные                                         | Видеоархивы            |

## 3. Функциональные требования

| ID    | Требование                                                                 | Приоритет |
|------:|---------------------------------------------------------------------------|----------:|
| FR-1  | Пользователь может просматривать расписание                              | P0 |
| FR-2  | Отображение залов и времени                                               | P0 |
| FR-3  | Спикер может создать доклад                                               | P0 |
| FR-4  | Спикер может редактировать доклад                                         | P0 |
| FR-5  | Спикер может загрузить слайды                                             | P0 |
| FR-6  | Участник может голосовать за доклад                                       | P0 |
| FR-7  | Организатор может изменять расписание                                     | P0 |
| FR-8  | Система отправляет уведомления об изменениях                              | P0 |
| FR-9  | Пользователь может подписаться на уведомления                             | P0 |
| FR-10 | Доступ к слайдам только для участников                                    | P0 |
| FR-11 | Поддержка нескольких конференций                                          | P0 |
| FR-12 | Брендинг конференции                                                      | P1 |
| FR-13 | Система оценок (web/email)                                                | P1 |
| FR-14 | Поддержка SMS-оценок                                                      | P2 |

## 4. Нефункциональные требования

| Требование                          | Метрика/основание                            |
|-----------------------------------|----------------------------------------------|
| Высокая доступность               | 99.9% уровень допущения                      |
| Обновление расписания             | < 1–2 сек                                    |
| Доставка уведомлений              | < 5 сек                                      |
| Контроль доступа к слайдам        | Аутентификация                               |
| Масштабируемость                  | Горизонтальная                               |
| Актуальность расписания           | Eventual consistency допустима               |
| Хранение файлов                   | До 100 МБ/файл (типичный размер презентаций) |
| Голосования                       | Высокая write-нагрузка                       |
| Поддержка языков                  | RU/EN минимум                                |

## 5. Ограничения

| Ограничение                        | Влияние на архитектуру            |
|------------------------------------|-----------------------------------|
| РФ, 2026 (регуляции)               | Хранение данных в РФ              |
| Персональные данные                | Соответствие ФЗ-152               |
| Email/SMS провайдеры               | Ограниченный выбор                |
| Push-уведомления                   | Зависимость от платформ           |
| Офлайн недоступен                  | Требуется онлайн в первую очередь |
| Разные конференции                 | Multi-tenant архитектура          |
| Доступ к слайдам только участникам | Аутентификация и вайтлист         |

## 6. Расчёт нагрузки

### Метрики

| Метрика                          |   Значение | Обоснование                                                |
|---------------------------------|-----------:|------------------------------------------------------------|
| Количество конференций          |   50 / год | Допущение                                                  |
| Участников на конференцию       |      5 000 | Крупные конференции часто имеют 1000–5000+ участников [^2] |
| Спикеров на конференцию         |        200 | Допущение                                                  |
| Всего пользователей             |    250 000 | Расчёт                                                     |
| DAU (в дни конференции)         |      5 000 | Внедрение приложения 50–90% [^3]                           |
| Пиковый онлайн                  |      2 000 | 30–50% одновременных пользователей                         |
| Голосов на пользователя         |         10 | Допущение                                                  |
| Уведомлений на пользователя     |       5–10 | Допущение                                                  |

---

### Расчёты

| Расчёт                         | Формула       |               Результат | Комментарий                               |
|--------------------------------|---------------|------------------------:|-------------------------------------------|
| RPS (просмотр расписания)      | 2000 / 10 сек |                 200 RPS | Пиковый сценарий                          |
| Голосования                    | 5000 × 10     |                  50 000 | Оценка                                    |
| RPS голосования                | 50 000 / 3600 |                  14 RPS | Средняя нагрузка                          |
| Уведомления                    | 5000 × 10     |                  50 000 | Broadcast                                 |
| Push throughput                | 50 000 / 60   | 833 сообщений в секунду | Взрывная нагрузка                         |
| Размер слайдов                 | 200 × 20MB    |             4 GB / конф | Типичные размеры PDF                      |
| Годовое хранилище              | 50 × 4GB      |                  200 GB | Без репликации                            |
| Пиковая нагрузка API           | 200 RPS × 1.5 |                 300 RPS | Соответствует нагрузке 2000 пользователей |
